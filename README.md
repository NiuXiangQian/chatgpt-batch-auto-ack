## ChatGPT Batch Question Assistant

一个用于 **批量向 ChatGPT 提问并自动收集答案** 的浏览器扩展侧边栏工具。

扩展会自动在 ChatGPT 页面中输入问题、提交、监控回答流（SSE），并将结果回传到侧边栏中进行展示、统计和导出，也支持将结果回调到你的自有接口。

---

### 功能特性

- **批量提问**：在侧边栏一次性输入多行问题，逐条自动发送给 ChatGPT。
- **系统命令前缀**：支持配置“系统命令”（例如角色设定、回答格式），自动拼接到每个问题前面。
- **执行控制**：开始 / 暂停 / 恢复 / 停止批量执行，支持失败任务一键重试。
- **进度与统计**：实时显示总数、已完成、成功、失败数量和整体进度百分比。
- **问题管理**：
  - 问题列表可拖拽排序；
  - 支持单条查看详情、编辑、从队列中移除；
  - 支持清空全部。
- **结果查看与导出**：
  - 在侧边栏查看每条问题的答案与引用来源（如有）；
  - 支持将当前结果导出为 JSON 文件。
- **Web 搜索（可选）**：在 ChatGPT 页面尝试自动开启网页搜索模式（根据当前界面自动识别）。
- **结果回调 API（可选）**：为每条完成的问题向你配置的 HTTP 接口发送 JSON 回调。
- **本地持久化**：问题队列及部分运行参数存储在 `chrome.storage.local` 中，下次打开仍可继续使用。

---

### 目录结构（核心文件）

- `manifest.json`：扩展清单（权限、入口文件配置等）。
- `sidepanel.html`：扩展侧边栏页面 UI。
- `sidepanel.css`：侧边栏样式。
- `sidepanel.js`：侧边栏主逻辑（问题队列、执行控制、UI 更新、本地存储等）。
- `sidepanel-extra.js`：侧边栏补充交互/功能逻辑。
- `line-numbers.js`：为问题输入框显示行号。
- `content.js`：注入到 ChatGPT 页面，用于：
  - 注入 `injected.js` 拦截 SSE；
  - 自动输入/提交问题；
  - 等待并收集回答流，调用后端解析；
  - 将结果通过 `chrome.runtime.sendMessage` 回传给侧边栏，并（如配置）发往外部 API。
- `injected.js`：在页面环境中重写 `window.fetch`，拦截 `/backend-api/conversation` 等接口的 SSE 数据，并通过 `window.postMessage` 推给 `content.js`。
- `config.js`：基础配置，例如：
  - `DEFAULT_SERVER_URL`：后端服务地址（用于解析 SSE 流）。
- `_locales/*/messages.json`：多语言文案（中/英/日/韩等）。

### 安装方式（开发者模式）

1. 克隆或下载本仓库代码到本地，例如：
   - `~/batch-auto-chatgpt-ack`
2. 打开 Chrome / Edge 浏览器，进入扩展管理页面：
   - Chrome：地址栏输入 `chrome://extensions/`
3. 开启右上角 **“开发者模式”**（Developer mode）。
4. 点击 **“加载已解压的扩展程序”**（Load unpacked）。
5. 选择本项目目录（包含 `manifest.json` 的文件夹）。
6. 加载成功后，在浏览器工具栏或侧边栏处打开扩展面板即可看到本插件。

### 使用说明

1. **打开 ChatGPT 官网**  
   在浏览器中登录 ChatGPT（确保可以正常对话）。

2. **打开扩展侧边栏**  
   在浏览器中打开本扩展的侧边栏页面（通常点击扩展图标 / 侧边栏图标即可）。

3. **配置系统命令（可选）**  
   在顶部“系统命令（每题前缀）”中填写统一的指令，例如：
   > 你是一名严谨的技术文档撰写助手，请用 Markdown 要点形式回答，并在最后给出总结结论。

4. **输入问题列表**  
   在“问题列表（每行一题）”输入框中，每行写一个要提交给 ChatGPT 的问题。

5. **点击“添加问题”**  
   点击“添加问题”按钮，问题会进入下方的队列列表，可拖拽排序、编辑或删除。

6. **开始批量执行**  
   在“执行控制”区域点击 **“开始执行”**：
   - 扩展会自动定位到 ChatGPT 输入框；
   - 按顺序输入问题（带上系统命令前缀）并提交；
   - 拦截并收集 SSE 流，将答案解析后回填到队列中。

7. **监控进度与日志**  
   - 在“进度条”和统计卡片中查看整体进度；
   - 在“执行日志”中查看每一步操作、错误与提示信息。

8. **导出结果或清空队列**  
   - 点击“导出结果（JSON）”按钮，可下载当前所有问题与答案；
   - 点击“清空全部”可以清除当前队列（需确认）。

### 结果回调 API（可选）

在侧边栏“更多设置”中可以配置 **“结果接收 API 地址”**（例如：`https://your-api.com/path`）。

当配置了合法的 HTTP/HTTPS 地址后，每当单个问题完成时，扩展会向该地址发送一次 `POST` 请求，`Content-Type: application/json`，请求体结构类似：

```json
{
  "question": "问题原文",
  "status": "completed or failed",
  "answer": "回答文本（失败时可能为空）",
  "timestamp": 1700000000000,
  "error": "失败原因（成功时为 null）"
}
```

> 注意：网络或接口错误不会中断扩展内部的正常执行，仅影响回调是否成功。

### 后端依赖说明

- 本扩展默认使用 `config.js` 中配置的后端地址：
  - `DEFAULT_SERVER_URL = "https://ai-assistant-api.aimegatron.com"`
- `content.js` 会将收集到的 SSE 片段发送到  
  `POST ${DEFAULT_SERVER_URL}/api/sse/parse`  
  由后端解析出最终回答、引用来源等结构化结果后再回填到侧边栏。
- 如果你自建后端，需要实现兼容的 SSE 解析接口（根据你的需求返回 `answer` / `sources` / `references` 等字段）。

---

### 国际化（i18n）

项目使用 Chrome 扩展内置的 `chrome.i18n` 多语言机制，在 `_locales` 目录中提供了多种语言的 `messages.json`。  
`sidepanel.js` 中的 `KEY_MAP` 会将文案 key 映射到 DOM 中的 `data-i18n` 属性，自动根据浏览器语言加载对应翻译。

---

### 已知限制与注意事项

- 本扩展依赖 ChatGPT 网页当前 DOM 结构和接口路径，若 OpenAI 改版，可能需要更新选择器和 SSE 拦截逻辑。
- 浏览器需要允许扩展在 `chat.openai.com`（或相关域名）上运行脚本。
- 大规模长时间批量提问可能触发 ChatGPT 的访问频率限制或风控，请自行控制节奏与问题数量。
